## 1. 设计目标

构建一个基于 Web 界面的操作系统智能代理，用户可通过自然语言（支持语音）对 Linux 服务器执行管理操作，实现"去命令行化"的系统管理体验。

### 核心能力
- 自然语言意图解析与系统操作执行
- 高风险操作识别与风险预警
- 多轮对话与连续任务编排
- 可视化结果反馈与语音交互

## 3. 数据流设计

### 3.1 正常操作流程

```
用户输入(自然语言)
    │
    ▼
┌──────────────┐
│ 意图解析      │ ── DeepSeek API 函数调用
│ (LLM)        │    输出: 工具名 + 参数

└──────┬───────┘
       │
   ┌───┴───┐
   │       │
   ▼       ▼
安全     危险
│        │
▼        ▼
执行     弹窗确认
│        │
▼        ▼
结果反馈  执行/拒绝
│        │
└────┬───┘
     ▼
自然语言反馈用户
```

### 3.2 多轮对话流程

```
用户: "查询当前磁盘使用情况"
Agent: 执行 df -h，返回格式化结果
用户: "那内存呢？"
Agent: 利用上下文识别"查询内存"，执行 free -h
用户: "清理一下缓存"
Agent: 识别为连续任务，执行 sync; echo 3 > /proc/sys/vm/drop_caches
```

## 4. 功能设计

### 4.1 基础系统管理（必须实现）

| 功能 | 命令示例 | 自然语言示例 |
|------|----------|-------------|
| 磁盘监测 | `df -h`, `du -sh /path` | "查看磁盘剩余空间" |
| 文件检索 | `find / -name "*.log"` | "查找所有日志文件" |
| 进程查询 | `ps aux`, `netstat -tlnp` | "查看占用80端口的进程" |
| 用户管理 | `useradd`, `userdel` | "创建一个新用户test" |

### 4.2 风险评估策略

**风险等级定义：**
- **SAFE** (绿色): 只读查询操作
- **WARNING** (黄色): 影响范围可控的修改
- **DANGER** (红色): 可能导致系统不可用的操作

**风险规则：**
```javascript
const RISK_RULES = {
  DANGER: [
    /rm -rf \//,                    // 删除根目录
    /rm -rf .*\/boot/,             // 删除boot目录
    /mkfs\..*/,                     // 格式化磁盘
    /dd if=.* of=\/dev\//,          // 直接写磁盘
    /chmod .* 777 .*\/etc/,        // 修改系统目录权限
    /userdel .*root/,               // 删除root用户
  ],
  WARNING: [
    /rm -rf/,                       // 强制递归删除
    /userdel/,                      // 删除用户
    /chmod 777/,                    // 开放全部权限
    /systemctl (stop|restart)/,     // 停止/重启服务
  ]
}
```

### 4.3 工具定义（Function Calling）

DeepSeek 函数调用 schema：

```json
{
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "execute_command",
        "description": "在Linux服务器上执行系统命令",
        "parameters": {
          "type": "object",
          "properties": {
            "command": {
              "type": "string",
              "description": "要执行的命令"
            },
            "explanation": {
              "type": "string",
              "description": "执行该命令的原因说明"
            }
          },
          "required": ["command", "explanation"]
        }
      }
    },
    {
      "type": "function",
      "function": {
        "name": "get_system_info",
        "description": "获取系统基本信息",
        "parameters": {
          "type": "object",
          "properties": {
            "info_type": {
              "type": "string",
              "enum": ["disk", "memory", "cpu", "network", "process"]
            }
          }
        }
      }
    }
  ]
}
```

## 8. 安全设计

### 8.1 命令白名单机制

默认只允许执行预定义的安全命令，危险命令需要额外确认。

### 8.2 沙箱执行

- 命令执行超时控制（默认30秒）
- 输出大小限制（防止洪水攻击）
- 敏感信息过滤（自动脱敏密码、密钥等）

### 8.3 审计日志

记录所有操作：时间、用户输入、执行命令、风险等级、执行结果。

## 9. 评分维度对应

| 评分维度 | 设计对应 |
|----------|----------|
| 基础需求执行 | 磁盘/文件/进程/用户管理功能实现 |
| 高风险识别与处置 | RiskEngine + 弹窗确认机制 |
| 复杂连续任务处理 | SessionManager 上下文维护 |
| 环境信息感知 | SystemDashboard 实时看板 |
| 基于环境的安全判断 | 结合当前OS类型动态调整命令 |
| 执行反馈清晰度 | 命令+结果+自然语言解释 |
| 交互反馈连贯性 | WebSocket实时推送 + 加载状态 |
| 多模态交互无劣变 | 语音输入支持 |
| 去命令行体验 | 纯自然语言交互，结果可视化 |
| 交互范式创新 | Agent自主决策，非简单透传 |
